# Необходимые доработки API (Backend TODO)

Для полной работы приложения Fit-Tracker необходимо реализовать следующие эндпоинты, которые сейчас отсутствуют или не описаны в OpenAPI:

## Тренировки (Workouts)
1. **DELETE `/workouts/{session_id}`**
   - Удаление записи о тренировке из истории.
2. **GET `/workouts/{session_id}/exercises`**
   - Получение списка упражнений, выполненных в рамках конкретной сессии.
3. **GET `/workouts/exercises/{workout_exercise_id}/sets`**
   - Получение списка подходов для конкретного упражнения в сессии.

## Программы и Планы (Programs & Plans)
4. **GET `/programs/{program_id}/plans`**
   - Получение списка планов (дней тренировок) для конкретной программы. Это критично для запуска тренировки по программе.
5. **DELETE `/programs/plans/{plan_id}`**
   - Удаление конкретного плана из программы.
6. **DELETE `/programs/plans/{plan_id}/exercises/{exercise_id}`**
   - Удаление упражнения из плана программы.

## Упражнения (Exercises)
7. **GET `/exercises/{exercise_id}`** (проверить работу)
   - Убедиться, что возвращаются полные данные об упражнении, включая `media_url` и описание.

---
**Примечание:** Приложение уже настроено на использование этих путей. Реализация этих эндпоинтов позволит пользователям удалять свои данные и просматривать историю тренировок.
